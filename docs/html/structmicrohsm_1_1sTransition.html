Detailed Description

Transition Struct.

Used to represent a transition from a source state with sourceID to a target state with targetID. Under UML there are three kinds of transitions: external/local and internal. This is represented with the kind field. Finally, transitions can execute so called effects, which will be triggered when the transition taken. When no effect is desired, the effect field can be left as nullptr
